# Shopee 连接器部署指南

## 版本信息

- **版本**: v2.0
- **描述**: 同步 Shopee 订单、产品和客户数据，支持多店铺管理和实时 Webhook 事件订阅
- **分类**: E-commerce

## 环境要求

### 1. 基础环境

- **Java**: JDK 17
- **Maven**: 3.8+
- **PostgreSQL**: 15+
- **Nacos**: 2.5.1+
- **Docker**: 20.10+ (可选)
- **Docker Compose**: 2.0+ (可选)

### 2. 硬件要求

- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **磁盘**: 20GB 以上

## 部署步骤

### 1. 安装 PostgreSQL 数据库

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql-15

# CentOS/RHEL
sudo yum install postgresql15-server
sudo postgresql-setup initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 2. 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库
CREATE DATABASE livepulse_shopee;

# 创建用户
CREATE USER shopee_user WITH PASSWORD 'your_password';

# 授权
GRANT ALL PRIVILEGES ON DATABASE livepulse_shopee TO shopee_user;

# 退出
\q
```

### 3. 执行数据库脚本

```bash
# 进入 SQL 脚本目录
cd livepulse-connector-shopee-open/docs/sql

# 执行建表脚本
psql -U shopee_user -d livepulse_shopee -f shopee_schema.sql
```

### 4. 配置 Nacos

#### 4.1 导入配置

在 Nacos 控制台创建以下配置：

**Data ID**: `postgresql-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://postgresql-host:5432/livepulse_shopee
    username: shopee_user
    password: your_password
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      connection-timeout: 30000
      max-lifetime: 1200000

mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.cs.entity
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

**Data ID**: `kafka-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  kafka:
    bootstrap-servers: kafka-host:9092
    consumer:
      group-id: shopee-connector-group
      auto-offset-reset: latest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### 5. 构建项目

```bash
# 进入项目根目录
cd livepluse-connectorx

# 编译打包
mvn clean package -DskipTests

# 查看生成的 JAR 文件
ls -lh livepulse-connector-shopee-open/target/livepulse-connector-shopee-open.jar
```

### 6. 配置应用

创建 `application-prod.yml` 文件：

```yaml
server:
  port: 23011

spring:
  application:
    name: livepulse-connector-shopee-open

# Shopee 定时任务配置
shopee:
  scheduled:
    enabled: true
    order-sync:
      cron: "0 0 */4 * * ?"  # 每4小时执行一次
    product-sync:
      cron: "0 0 2 * * ?"  # 每天凌晨2点
    customer-sync:
      cron: "0 0 3 * * ?"  # 每天凌晨3点
```

### 7. 启动应用

#### 方式一：直接启动

```bash
java -jar livepulse-connector-shopee-open/target/livepulse-connector-shopee-open.jar \
  --spring.profiles.active=prod \
  --NACOS_SERVER_ADDR=nacos-host:8848
```

#### 方式二：使用 Docker 启动

```bash
# 构建镜像
cd livepulse-connector-shopee-open
docker build -t livepulse-connector-shopee:2.0 .

# 启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 8. 配置 Nginx（可选）

```nginx
upstream shopee-connector {
    server localhost:23011;
}

server {
    listen 80;
    server_name shopee.yourdomain.com;

    location / {
        proxy_pass http://shopee-connector;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Shopee API 配置

### 1. 创建 Shopee 开放平台应用

1. 访问 [Shopee Open Platform](https://open.shopee.com/)
2. 创建新的应用
3. 获取 Partner ID 和 Partner Key
4. 配置回调 URL

### 2. 获取访问令牌

1. 使用 OAuth 2.0 流程获取授权码
2. 使用授权码换取访问令牌
3. 定期刷新访问令牌

### 3. 配置 Shopee 店铺连接

通过 API 添加 Shopee 店铺配置：

```bash
curl -X POST http://localhost:23011/shopee/shop \
  -H "Content-Type: application/json" \
  -d '{
    "shopId": 123456789,
    "shopName": "My Shop",
    "region": "TW",
    "accessToken": "your-access-token-here",
    "partnerId": 987654321,
    "partnerKey": "your-partner-key-here",
    "enabled": true
  }'
```

## 故障排查

### 1. 检查日志

```bash
# Docker 日志
docker-compose logs -f

# 应用日志
tail -f logs/livepulse-connector-shopee-open.log
```

### 2. 常见问题

#### 问题 1：无法连接到 PostgreSQL

检查数据库连接配置：

```bash
psql -U shopee_user -d livepulse_shopee -h postgresql-host
```

#### 问题 2：无法注册到 Nacos

检查 Nacos 地址配置：

```bash
curl http://nacos-host:8848/nacos/v1/ns/instance/list?serviceName=shopee-open-connector-server
```

#### 问题 3：Shopee API 调用失败

检查 API 凭据配置：
- 确认访问令牌有效
- 确认 Partner ID 和 Partner Key 正确
- 确认店铺 ID 格式正确

## 数据模型说明

### 核心表结构

1. **shopee_shop**: Shopee 店铺配置
2. **shopee_order**: 订单数据
3. **shopee_product**: 产品数据
4. **shopee_customer**: 客户数据
5. **shopee_webhook_log**: Webhook 日志
6. **shopee_sync_log**: 同步日志

### 同步频率

- 订单：每4小时同步最近30天数据
- 产品：每天凌晨2点同步最近30天数据
- 客户：每天凌晨3点同步最近30天数据

## 安全建议

1. **敏感数据加密**：对访问令牌进行加密存储
2. **访问控制**：配置 Nginx IP 白名单
3. **HTTPS 配置**：生产环境使用 SSL 证书
4. **日志脱敏**：避免在日志中输出敏感信息
5. **令牌轮换**：定期更新访问令牌
